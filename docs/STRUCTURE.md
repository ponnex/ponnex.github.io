# Sitemap & canonical structure

Generated/maintained via `/seo sitemap generate`.

## Canonical + og:url

Emitted **globally, per-route** in `app/app.vue` (`rel=canonical` + `og:url`),
normalized to the **trailing-slash** form. Individual pages set their own
`title` / `description` / `ogImage` / JSON-LD but delegate canonical + og:url to
`app.vue` so there is a single source of truth. The old hardcoded `og:url` in
`nuxt.config.ts` (which pointed every page at the site root) was removed.

Why trailing slash: the prerendered output is directory-style
(`/projects/sia/index.html`), so the live host serves `/projects/sia/` as the
`200` and 301-redirects the no-slash form to it. Canonicals + sitemap `<loc>`
must point at the `200` URL, not the redirecting one.

## Sitemap

Generated by **`@nuxtjs/sitemap`** (registered in `nuxt.config.ts`), prerendered
to `/sitemap.xml` at build time and served at `https://ponnex.dev/sitemap.xml`.
Referenced from `public/robots.txt`.

Config (`nuxt.config.ts`):
- `site: { url: 'https://ponnex.dev', trailingSlash: true }` — canonical domain;
  trailing-slash matches the directory-style `200` output.
- `sitemap.exclude: ['/thankyou', '/thankyou/']` — noindex confirmation page.
- `sitemap.urls: ['/ramos_resume.pdf']` — the PDF is a static file, not a route,
  so it is added explicitly.

The module **auto-discovers every prerendered route**: `nitro.prerender.crawlLinks`
reaches all project slugs from `/`, so adding a project to `app/data/projects.ts`
flows into the sitemap automatically — no manual list to maintain.

(the module emits none for prerendered routes; omitted is cleaner than an
all-identical build-time stamp). The module also auto-includes `<image:image>`
entries for project images it discovers in the page HTML.

## Maintenance

Auto-maintained — adding a project to `app/data/projects.ts` (and linking it from
`/projects/`, which the index already does) adds it to the sitemap on the next
build. No manual editing needed. Re-run `npm run generate` to regenerate.

prerendered HTML stays fully visible for crawlers/no-JS.
